Bienvenida

¡Bienvenido al segundo curso de tu camino por el mundo de las redes! En este módulo aprenderás a configurar redes de manera básica y comprenderás cómo los protocolos hacen posible la comunicación entre dispositivos. Este conocimiento es esencial para desempeñarte en escenarios reales donde se requiere implementar, modificar o diagnosticar redes. Prepárate para conectar teoría con práctica.

Módulo 1: ¿Qué es una dirección IP?

Pregunta: ¿Qué es una dirección IP?

Respuesta: Una dirección IP (Internet Protocol) es un identificador único que se asigna a cada dispositivo en una red para que pueda comunicarse con otros. Se compone de una serie de números separados por puntos (IPv4) o por dos puntos (IPv6).

Por ejemplo, en IPv4 una dirección puede ser 192.168.0.1. Está dividida en dos partes:

  • Parte de red: Identifica a qué red pertenece el dispositivo.
  • Parte de host: Identifica el dispositivo dentro de esa red.

Las direcciones IP pueden ser públicas o privadas, y estáticas o dinámicas. Son fundamentales para el enrutamiento y entrega de datos en redes locales e Internet.

Módulo 2: Configuración de red IP manual y automática

Pregunta: ¿Cómo se configura una dirección IP?

Respuesta: La configuración puede hacerse de dos formas:

  • Manual (estática): El usuario asigna una IP fija a un dispositivo. Se usa cuando se necesita una dirección constante, como en servidores.
  • Automática (DHCP): Un servidor DHCP asigna IPs automáticamente a los dispositivos dentro de la red. Esto facilita la gestión de múltiples equipos.

Además de la IP, también se deben configurar otros parámetros como:

  • Máscara de subred: Define qué parte de la IP pertenece a la red y cuál al host.
  • Puerta de enlace (Gateway): Es la IP del dispositivo que conecta la red local con otras redes, generalmente un router.
  • DNS: Son servidores que traducen nombres de dominio (como google.com) en direcciones IP.

Módulo 3: Protocolo TCP/IP

Pregunta: ¿Qué es el protocolo TCP/IP?

Respuesta: TCP/IP es un conjunto de protocolos que permite la comunicación entre computadoras a través de redes. Se basa en un modelo de cuatro capas:

  • Capa de aplicación: Donde se ejecutan los programas (HTTP, FTP, DNS).
  • Capa de transporte: Proporciona comunicación confiable (TCP) o rápida (UDP).
  • Capa de Internet: Encargada de direccionar y enrutar los paquetes (IP).
  • Capa de acceso a red: Se relaciona con el hardware físico (Ethernet, Wi-Fi).

Este protocolo es la base de Internet y redes locales, garantizando que los datos lleguen correctamente a su destino.

Módulo 4: Configuración básica de un router doméstico

Pregunta: ¿Cómo se configura un router básico en casa?

Respuesta: Para configurar un router doméstico:

  1. Conéctate al router mediante un navegador con la dirección IP del router (como 192.168.1.1).
  2. Ingresa las credenciales de administrador (normalmente vienen en una etiqueta del router).
  3. Cambia el nombre de la red (SSID) y configura una contraseña segura.
  4. Activa el cifrado WPA2 o WPA3 para la seguridad Wi-Fi.
  5. Configura parámetros como el canal, DHCP y control parental si es necesario.

La configuración básica ayuda a proteger tu red doméstica y optimizar la señal.

Módulo 5: Puertos y protocolos comunes

Pregunta: ¿Qué son los puertos y protocolos en redes?

Respuesta: Los puertos son puntos virtuales que permiten a los dispositivos identificar a qué servicio deben enviar o recibir datos. Cada protocolo utiliza puertos específicos. Ejemplos comunes:

  • HTTP - Puerto 80: Para navegación web sin cifrado.
  • HTTPS - Puerto 443: Para navegación segura.
  • FTP - Puerto 21: Para transferencia de archivos.
  • SSH - Puerto 22: Para acceso remoto seguro.
  • DNS - Puerto 53: Para resolver nombres de dominio.

Conocer estos puertos ayuda a configurar firewalls, routers y servicios de forma eficiente.

Módulo 6: Comandos de red básicos

Pregunta: ¿Qué comandos se usan para diagnosticar redes?

Respuesta: Existen comandos esenciales para diagnosticar y verificar el estado de una red. Algunos son:

  • ping: Verifica si otro dispositivo responde (ej. ping google.com).
  • ipconfig (Windows) / ifconfig (Linux): Muestra la configuración de IP del dispositivo.
  • tracert / traceroute: Muestra la ruta que siguen los paquetes hasta un destino.
  • netstat: Muestra conexiones activas y puertos utilizados.
  • nslookup: Consulta información de DNS.

Estos comandos son la base del diagnóstico de conectividad en entornos domésticos o empresariales.
